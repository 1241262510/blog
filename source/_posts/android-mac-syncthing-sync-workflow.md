## 手机与 Mac 之间，截图怎么传更高效

手机和电脑之间传图、传截图，看起来是小事，但一旦变成高频动作，效率差别就会非常明显。

常见办法大致有这几种：

- 微信文件传输助手
- Edge `Drop`
- `Syncthing`
- 其他同步工具

前两种都能用，但如果是长期高频同步，我最后最推荐的还是 `Syncthing`。

## 为什么重点推荐 Syncthing

最开始我对 `Syncthing` 没什么深刻印象，甚至一度觉得不如 `Drop` 来得直接。上次折腾它，还是在 NAS 上同步手机文件，后来就没有继续深入使用。

但这次重新实践下来，结论完全变了。

如果只是偶尔传一两张图，`Drop` 确实够用；但如果截图、文档、图片需要频繁同步到 Mac，`Syncthing` 的体验会明显更高效。

以截图为例，两种工作流对比很直观：

- `Drop`：截图，复制或选中，发送到 Drop，Mac 端接收
- `Syncthing`：截图，接收

差别就在这里。前者依赖一次次手动操作，后者更接近自动同步。

## 整体流程

整个搭建过程并不复杂，主要就是四步：

- 安装软件
- 设置设备 ID
- 设置同步目录
- 开始同步

## 软件安装与基本设置

### Mac 端

Mac 上可以直接通过 `brew` 安装：

```bash
brew install syncthing
```

安装完成后运行：

```bash
syncthing
```

软件启动后，会提供一个本地 GUI 管理页面：

```text
http://127.0.0.1:8384
```

下面这张图就是 `Syncthing` 在 Mac 端的管理界面：

![](/assets/1e6629380d08cecb5b971edffc995384ab19de5a.png)

在 Mac 端 GUI 页面里，进入“操作”“设置”，可以继续调整默认值。

![](/assets/f16b25907b773410d3ba00ddd4a7aeea1d38b4fe.png)

“编辑文件夹默认值”里，建议把 Mac 端的文件夹类型设成“只接收”。这样同步方向就是从手机到 Mac，逻辑会更清晰。

![](/assets/4977b82a64439c32e5264bc04ae551d55cdf82e5.png)

常规选项大多保持默认即可。

![](/assets/ffb7c09528e61c49a129be39460e00c94827d1e6.png)

这里最重要的是“文件夹路径”，也就是以后 Mac 用来接收手机截图或文件的目录。

![](/assets/bfce0d71b611e59a5c6c544a163ceb8931595b04.png)

比如手机上刚截了一张图，同步完成后，这张图就应该立刻出现在 Mac 指定的目录里。

![](/assets/7c3302eade017e781931ab2916d7fc92a812dafb.png)

设备默认值也可以顺手改一下名字，比如改成 `mac mini` 或者自己更容易识别的设备名，后面手机和 Mac 配对时会直观很多。

![](/assets/bbe75b14fcbcaf5efc5c377cebe676b8495aa1bb.png)

在主界面中，还可以继续添加更多同步文件夹。

## 手机端设置

### 安装 Syncthing

安卓端应用商店里就能搜到 `Syncthing` 或 `Syncthing-Fork`。

![](/assets/71be92d66b7f341994eb8fc4deb49039b5a612bf.png)

如果是新配置，可以直接点“+”创建同步文件夹。

![](/assets/d2f433f417276cec8aab8e2fdde6b3f0cb06523a.png)

和 Mac 端类似，手机端也需要设置同步目录位置。同步模式建议设成“只发送”，其他选项一般保持默认即可。

![](/assets/33081c11cc2ed22c49addabcf6e4c21ce4f96765.png)

我手机上的截图目录就在这里。大多数安卓手机的截图目录一般在下面两个位置之一：

- `Pictures/Screenshots`
- `DCIM/Screenshots`

具体路径还是要以自己手机的实际情况为准。目录确定之后，后面的同步就会轻松很多。

## 设备配对

同步目录设置好之后，还需要把手机和 Mac 配对起来。

在手机端，从左侧菜单进入“显示设备 ID”，这里会显示二维码和设备信息。

![](/assets/726c744b1e3a3ac37821ef387c5656dcae5d67e9.png)

更顺手的方式是：Mac 显示二维码，手机直接扫码。

### Mac 端操作

Mac 主界面进入“操作”“显示 ID”。

![](/assets/8266fa0f2edd4d91e004394cc5992dec40e74c7e.png)

![](/assets/03fb1c61b7efbe71c627758fb3354b844a127973.png)

这就是 Mac 端的二维码，直接用手机扫码就可以完成配对。

### 手机端操作

手机端进入“设备”，开始进行硬件链接。

![](/assets/50379fd7e733751970efb43c0db078bfb74677be.png)

随后扫码完成设备连接。

做完这一步之后，手机和 Mac 就可以开始单向同步了，手机端的数据会自动进入 Mac 的接收目录。

## 总结

看起来配置步骤稍微有一点多，但真正核心的工作其实只有三件事：

- 设置同步目录
- 设置设备连接
- 把同步方向配置清楚

其余选项大多可以保持默认。

如果你用的是苹果手机，系统本身的同步能力已经比较强；但如果是安卓和 Mac 的组合，`Syncthing` 的灵活性会更明显。

它的优势不只是“能同步”，而是同步动作快、自动化程度高，而且适配范围非常广。

尤其是截图、文档、图片这种高频小文件，体验提升会非常明显。真正常用起来之后，会发现同步几乎是按秒发生的。
