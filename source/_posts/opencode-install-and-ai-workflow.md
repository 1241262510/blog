## 摘要

`OpenCode` 是一个集成多种 AI 代理能力的协作平台，既有免费模型，也有付费模型。对普通用户来说，它的价值不只是“多一个工具”，而是让很多原本需要自己查教程、跑命令、反复试错的软件安装过程，开始变得更像自然语言协作。

## OpenCode 的安装

`OpenCode` 是一个协作软件平台，集成了多种 AI 代理。免费的、收费的，都可以在同一个工具链里找到。

这里直接使用官网提供的安装命令进行安装。以我自己的使用场景为例，我是在 Linux 虚拟机里完成安装。

安装命令：

```bash
curl -fsSL https://opencode.ai/install | bash
```

软件界面：

![](https://files.mdnice.com/user/113193/7be8677a-60b2-4a1e-a279-ca793e4d3f04.png)

进入之后，输入指令 `/models`，就可以切换不同的 AI 模型。

![](https://files.mdnice.com/user/113193/46d3a72d-90b5-4bbd-9822-9a2baf1308d0.png)

可以先试试几个免费的模型，感受一下实际效果。我这里已经配置好了 `GLM 4.7 Code Plan`，也可以直接在这里调用。

## 软件安装的变化

以前自己安装软件，大致是这样一个循环：

- 搜索教程
- 复制和粘贴安装代码
- 看实际输出结果
- 如果不成功，就继续重复这个循环，直到成功

有了 AI 之后，现在的安装流程已经变得很不一样。

- 先准备必要依赖，例如：

```bash
sudo apt install curl
```

- 安装 `OpenCode`，再选一个免费模型
- 直接对 AI 提自然语言要求，让它协助完成安装

通过这个简单对比就能看出来，现在的软件安装方式和过去已经有了明显差异。说是变革也许还可以再观察，但变化已经非常清楚了。

## 技术门槛正在下降

这类变化最直接的影响，就是技术门槛在快速下降。

以前安装软件，至少要求你会一些基础工具，例如 `vim`、`nano`，或者最少也要能完成常见的编辑操作。增删改查、改配置、调整参数，这些几乎都是默认要求。

现在需要的是什么？很多时候，会说自然语言就够了。

比如安装 `Docker` 和 `Docker Compose`。

过去通常是：先上搜索引擎查文章、查命令、自己测试。

现在则更像这样：

`请帮我安装 docker 和 docker compose。`

然后 AI 代理开始执行、检查、修正、补齐，最后把事情做完。这个变化本身就已经足够惊人。

## 总结

回头再看，会发现 AI 不只是“帮你回答问题”，它正在默默改变人们使用电脑和操作软件的方式。

如果用一句更直接的话来概括，就是：

- 做得更多
- 速度更快
- 结果更整齐
- 试错成本更低

某种意义上，这已经很接近“数字员工”的工作方式了。

不是我们反应慢，而是这个世界确实变化得太快。
