## 摘要

`OpenCode` 是一个集成多种 AI 代理能力的协作平台，既有免费模型，也有付费模型。它真正有意思的地方，不只是把模型集中到一个入口，而是让原本依赖搜索、复制命令、反复试错的软件安装过程，开始变成一种更接近自然语言协作的体验。

## OpenCode 的安装

`OpenCode` 集成了多种 AI 代理能力，免费的、收费的模型都可以在同一个工具链里找到。

这里直接使用官网提供的命令安装。以我自己的使用场景为例，我是在 Linux 虚拟机里完成这一步。

安装命令：

```bash
curl -fsSL https://opencode.ai/install | bash
```

软件界面和模型选择部分，这里先不放无关截图，后面统一替换成你实际要保留的图片。

进入之后，输入 `/models`，就可以切换不同的 AI 模型。

可以先试试几个免费的模型，感受一下整体体验。我这里已经配置好了 `GLM 4.7 Code Plan`，也可以直接在这里调用。

## 软件安装的变化

以前自己安装软件，大致是这样一个循环：

- 搜索教程
- 复制和粘贴安装代码
- 看实际输出结果
- 如果不成功，就继续重复这个循环，直到成功

有了 AI 之后，现在的安装流程已经开始明显变化。

- 先准备必要依赖，例如：

```bash
sudo apt install curl
```

- 安装 `OpenCode`，再选一个可用模型
- 直接用自然语言告诉 AI 你要安装什么，让它协助完成安装

通过这个简单对比就能看出来，现在的软件安装方式和过去已经有了明显差异。是不是“变革”可以继续观察，但变化本身已经非常清楚。

## 技术门槛正在下降

这种变化带来的直接结果，就是技术门槛正在下降。

以前安装软件，至少要求你会一些基础工具，例如 `vim`、`nano`，或者最少也要能完成常见编辑操作。改配置、调参数、检查报错，这些几乎都是默认要求。

现在需要的是什么？很多时候，会说自然语言就够了。

比如安装 `Docker` 和 `Docker Compose`。

过去通常是：先上搜索引擎查文章、查命令、自己测试。

现在更像这样：

`请帮我安装 docker 和 docker compose。`

然后 AI 代理开始执行、检查、修正、补齐，最后把事情做完。真正令人意外的，不是它会回答问题，而是它开始能把一整段安装过程接过去。

## 总结

回头再看，会发现 AI 不只是“帮你回答问题”，它正在慢慢改变人们使用电脑、安装软件和解决问题的方式。

如果用一句更直接的话来概括，就是：

- 做得更多
- 速度更快
- 结果更整齐
- 试错成本更低

某种意义上，这已经开始接近“数字员工”的工作方式。

不是我们反应慢，而是这个世界确实变化得太快。
