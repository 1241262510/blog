自定义域名有很多玩法。最常见的一种，就是做一个临时邮箱或测试邮箱。很多免费服务、开发者活动，甚至像英伟达部分试用资源的申请，都可能要求使用自定义域名邮箱。

如果直接购买现成服务，常见方案有 Google Workspace 和 Zoho Mail。它们都很好用，但本质上都是付费服务。那如果只是想低成本测试一下，有没有别的办法？有，可以直接用开源项目配合 Cloudflare，搭一套自己的域名邮箱收件系统。

这篇文章记录一遍完整实操流程。

项目地址：

```text
https://github.com/maillab/cloud-mail
```

![](/assets/SCR-20260522-bzrd.png)

建议先把项目 fork 到自己的 GitHub 仓库，方便后续部署和管理。

![](/assets/SCR-20260522-cawy.png)

## 1. 在 Cloudflare 中部署项目

进入 Cloudflare 后，选择 `Workers & Pages`。

![](/assets/SCR-20260522-cbwt.png)

点击创建应用，选择通过 GitHub 关联仓库。

![](/assets/SCR-20260522-ccef.png)

![](/assets/SCR-20260522-cctv.png)

选择刚才 fork 的邮箱项目，开始部署。

![](/assets/SCR-20260522-cdjf.png)

![](/assets/SCR-20260522-cdta.png)

在高级设置中，有一个路径配置项，把它改成：

```text
/mail-worker
```

然后点击部署。

![](/assets/SCR-20260522-cgdt.png)

Cloudflare 会开始拉取项目、克隆仓库并自动部署。部署完成后，系统会先给你一个默认域名，后面可以再切换到自己的域名。

点击“添加域名”进行修改。

![](/assets/SCR-20260522-cgdt%201.png)

比如最终可以改成：

```text
mail.你的域名.xyz
```

例如我这里使用的是 `mail.***.xyz`。你实际操作时，把它替换成自己已经托管到 Cloudflare 的域名即可。

## 2. 配置环境变量

进入项目的 `Settings` 页面，开始添加环境变量。

![](/assets/SCR-20260522-clmf.png)

![](/assets/SCR-20260522-cmil.png)

首先添加 `domain` 变量：

- 类型：`JSON`
- 变量名：`domain`
- 值：

```json
["你的域名"]
```

注意这里使用的是英文符号。填写完成后点击部署。

![](/assets/SCR-20260522-cpjm.png)

继续添加 `admin` 变量：

- 类型：`Text`
- 变量名：`admin`
- 值：

```text
admin@你的域名
```

![](/assets/SCR-20260522-cqiy.png)

再继续添加 `jwt_secret` 变量：

- 类型：`Text`
- 变量名：`jwt_secret`
- 值：自己设置的一串密码

这个值建议设置得复杂一些，后面登录和初始化会用到。

## 3. 创建数据库

### 3.1 创建 D1 数据库

![](/assets/SCR-20260522-ctqs.png)

![](/assets/SCR-20260522-cupp.png)

先创建一个 D1 数据库，起一个名字，然后开始创建。

![](/assets/SCR-20260522-cvba.png)

### 3.2 创建 KV 数据库

接着进入 `Workers KV`，继续创建一个 KV 数据库。

后面需要把这两个数据库都绑定到当前项目中。

## 4. 绑定 D1 和 KV

回到项目页面，继续进行绑定设置。

![](/assets/SCR-20260522-cyaa.png)

![](/assets/SCR-20260522-cyob.png)

点击“绑定”。

![](/assets/SCR-20260522-cyxt.png)

先添加 KV 数据库绑定。

![](/assets/SCR-20260522-czhp.png)

![](/assets/SCR-20260522-czsd.png)

按照图中参数填写，并完成 KV 数据库的绑定。

![](/assets/SCR-20260522-dame.png)

然后再绑定 D1 数据库。

![](/assets/SCR-20260522-dawi.png)

![](/assets/SCR-20260522-dbls.png)

![](/assets/SCR-20260522-dbyj.png)

## 5. 配置邮件路由

回到账户首页，对域名进行邮件路由设置。

这里的目标是：所有接收到的邮件，都先交给 Worker 处理。

![](/assets/SCR-20260522-dgam.png)

![](/assets/SCR-20260522-dirq.png)

添加邮箱相关 DNS 记录时，点击“添加丢失的记录”即可。Cloudflare 一般会自动识别所需记录并保存。

## 6. 初始化测试

部署完成后，先访问下面这个初始化地址：

```text
http://mail.你的域名/api/init/密码
```

如果页面返回 `success`，说明初始化成功。

![](/assets/SCR-20260522-dmni.png)

![](/assets/SCR-20260522-dnmg.png)

接着访问登录地址：

```text
http://mail.你的域名/login
```

![](/assets/SCR-20260522-dokt.png)

注册 `admin` 邮箱后登录，就可以看到后台界面。

![](/assets/SCR-20260522-dpzm.png)

## 7. 当前测试结果

我这里简单测试了一下：

- 可以正常收邮件
- 暂时还不能发邮件

如果你的目标只是做一个低成本测试邮箱或者接收型邮箱，到这一步基本已经够用了。后面如果要继续完善发信能力，可以再补 SMTP 或其他外发方案。
