使用 Rime 输入法时，经常会遇到这样的情况：看到一个陌生字，既不知道读音，也不知道应该输入什么编码。

雾凇拼音内置了“部件拆字反查”功能。只要认识组成这个字的部件，就可以用部件的完整拼音查出目标字。

## 一、基本用法

反查格式为：

```text
uU + 部件的完整拼音
```

注意：

```text
uU
```

由小写字母 `u` 和大写字母 `U` 组成。

后面的编码使用完整拼音，不使用小鹤双拼，也不是小鹤音形码。

## 二、查询示例

### 查询“筛”

“筛”可以拆分为：

```text
竹 + 师
```

因此输入：

```text
uUzhushi
```

候选栏中会出现：

```text
筛  shai
```

查到读音 `shai` 后，如果平时使用小鹤双拼，可以换算为：

```text
shai = ud
```

以后直接输入：

```text
ud
```

即可打出“筛”。

### 查询“淼”

“淼”由三个“水”组成：

```text
水 + 水 + 水
```

输入：

```text
uUshuishuishui
```

候选栏中会出现：

```text
淼  miao
```

小鹤双拼编码为：

```text
miao = mn
```

### 查询“犇”

“犇”由三个“牛”组成：

```text
牛 + 牛 + 牛
```

输入：

```text
uUniuniuniu
```

候选栏中会出现：

```text
犇  ben
```

小鹤双拼编码为：

```text
ben = bf
```

## 三、推荐使用流程

如果平时使用小鹤双拼，可以按照下面的流程查询陌生字：

```text
看到陌生字
↓
观察它由哪些部件组成
↓
输入 uU + 部件完整拼音
↓
从候选栏中找到目标字
↓
查看目标字的完整拼音
↓
换算为小鹤双拼编码
```

例如：

```text
筛
↓
竹 + 师
↓
uUzhushi
↓
shai
↓
ud
```

## 四、为什么使用完整拼音反查

小鹤双拼适合日常快速输入，但遇到陌生字时，使用双拼反查有一个问题：如果记不住某个部件的双拼编码，仍然无法继续输入。

使用完整拼音更容易理解，也方便临时查询：

```text
竹 = zhu
师 = shi
水 = shui
牛 = niu
```

因此，比较实用的配置方式是：

- 日常输入：使用小鹤双拼。
- 查询陌生字：统一使用 `uU + 部件完整拼音`。
- 查到读音后：再换算成小鹤双拼编码。

## 五、如何将反查统一改为全拼

如果此前为小鹤双拼安装过拆字补丁，可能会出现下面的情况：

```text
uUzhushi
```

无法查询“筛”，反而必须输入：

```text
uUvuui
```

这是因为小鹤补丁修改了共享的拆字词典。雾凇全拼和小鹤双拼都引用同一个拆字词典，所以反查编码会一起变成双拼。

在 macOS 鼠须管中，打开：

```text
~/Library/Rime
```

删除：

```text
radical_pinyin.custom.yaml
```

然后点击菜单栏中的鼠须管图标，选择：

```text
重新部署
```

重新部署后，无论当前使用“雾凇拼音”还是“小鹤双拼”，都可以使用完整拼音反查：

```text
uUzhushi        → 筛
uUshuishuishui  → 淼
uUniuniuniu     → 犇
```

删除这个补丁不会影响日常的小鹤双拼输入。

## 六、生僻字显示为方框怎么办

部分生僻字可能显示为方框或问号。这不是反查失败，而是系统字体缺少对应字形。

可以安装覆盖生僻字较多的“遍黑体”：

```text
遍黑体 P1
遍黑体 P2
```

安装后，编辑：

```text
~/Library/Rime/squirrel.custom.yaml
```

添加：

```yaml
patch:
  style/font_face: "遍黑体 P1, 遍黑体 P2, PingFang SC"
```

重新部署鼠须管即可。

## 七、总结

雾凇拼音的拆字反查并不是 Rime 默认自带的功能，而是雾凇拼音预先集成的 `radical_pinyin` 部件拆字词典。

最值得记住的操作只有一条：

```text
uU + 部件的完整拼音
```

遇到不认识的字，不需要先知道读音。只要能拆出部件，就可以反查读音，再转换为自己熟悉的小鹤双拼编码。
