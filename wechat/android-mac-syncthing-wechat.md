# 安卓手机和 Mac 传截图，我最后留下了 Syncthing

很多时候，真正影响效率的，不是多大的任务，而是那些每天都会重复的小动作。

比如：

- 手机截一张图
- 想发到 Mac 上
- 再拖进文档、博客、聊天窗口里

偶尔做一次，这不算什么。  
但如果这个动作每天都要重复很多次，效率差别就会被迅速放大。

我前后试过几种方式：

- 微信文件传输助手
- Edge Drop
- Syncthing

最后真正留下来的，是 `Syncthing`。

## 为什么最后是 Syncthing

如果只是临时传一两张图片，`Drop` 其实够用。

但高频使用的时候，问题就会出来：

- 还是要手动发送
- 还是要等待接收
- 每次都要重复一遍动作

而 `Syncthing` 的感觉更像是：

- 手机截图
- Mac 自动出现

两者最大的区别，不是“能不能传”，而是有没有把中间那一段重复动作拿掉。

## 我现在的实际工作流

我现在这套流程非常简单：

1. 手机截图
2. `Syncthing` 自动同步
3. Mac 里的同步目录立刻收到图片
4. 直接拖去博客、文档或者聊天窗口

如果只是看步骤，似乎只少了一两步；但真正高频用起来，体感差别非常大。

## Mac 端怎么装

Mac 上我直接用 `brew` 安装：

```bash
brew install syncthing
```

安装完成后运行：

```bash
syncthing
```

启动之后，浏览器里打开：

```text
http://127.0.0.1:8384
```

就能进入它的管理界面。

![](https://1241262510.github.io/blog/assets/1e6629380d08cecb5b971edffc995384ab19de5a.png)

这里建议做两个关键设置：

- Mac 端文件夹模式设成“只接收”
- 给同步文件夹指定一个固定目录

这样逻辑最清楚：手机发，Mac 收。

![](https://1241262510.github.io/blog/assets/4977b82a64439c32e5264bc04ae551d55cdf82e5.png)

![](https://1241262510.github.io/blog/assets/bfce0d71b611e59a5c6c544a163ceb8931595b04.png)

## 安卓端怎么配

安卓端我装的是 `Syncthing-Fork`。

新建同步文件夹时，重点看两项：

- 同步目录位置
- 同步模式

目录一般是手机截图目录，例如：

- `Pictures/Screenshots`
- `DCIM/Screenshots`

手机端建议设成“只发送”。

![](https://1241262510.github.io/blog/assets/71be92d66b7f341994eb8fc4deb49039b5a612bf.png)

![](https://1241262510.github.io/blog/assets/33081c11cc2ed22c49addabcf6e4c21ce4f96765.png)

## 配对其实很简单

目录设好之后，接下来就是设备配对。

我觉得最顺手的方式是：

- Mac 显示二维码
- 手机直接扫码

Mac 端在“操作”里可以显示设备 ID 和二维码：

![](https://1241262510.github.io/blog/assets/8266fa0f2edd4d91e004394cc5992dec40e74c7e.png)

手机端进入“设备”页面后，扫码完成连接：

![](https://1241262510.github.io/blog/assets/50379fd7e733751970efb43c0db078bfb74677be.png)

做完这一步，手机和 Mac 就可以开始单向同步了。

## 这套方案最适合什么人

如果你是下面这种情况，我觉得它非常值得配：

- 经常要把手机截图发到电脑
- 安卓手机 + Mac 组合
- 会频繁写文档、写博客、发图
- 讨厌重复做“发送-接收”这段动作

如果你只是偶尔传一张图，其实 `Drop` 也能用。  
但只要进入高频场景，`Syncthing` 的优势会越来越明显。

## 最后

看起来配置步骤比直接传图要多一点，但真正配完以后，后面的体验会轻松很多。

对我来说，这类工具最重要的不是“功能多强”，而是它能不能把重复动作从工作流里拿掉。

`Syncthing` 做到的，正是这一点。
