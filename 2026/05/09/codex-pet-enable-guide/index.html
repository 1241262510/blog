Codex 宠物开启实操：终端命令不行，桌面端可以

开启 Codex 宠物

一开始我以为这个功能会比较简单，和 Claude Code 在终端里的设置方式类似，直接输入指令：

应该就可以开启。

但实际操作之后，发现并不是这么回事。在 Codex 的终端里输入这个指令，没有任何反应。

看来还是要走更稳妥的路径：

  • 下载 Codex 桌面端
  • 在 App 中开启宠物功能

实操下来，这条路径是可行的。

桌面端

访问 Codex 官网，直接下载桌面端。

我使用的是 Mac 操作系统，所以选择 macOS 版本下载。

安装并启动之后，系统会提示是否从 Claude 迁移数据。我这里选择了“是”，迁移了一部分数据。如果你的系统里没有 Claude 相关数据，也可以直接跳过。

安装完成后的界面比较朴素。

设置

选择“设置”。

进入设置后，选择“外观”。

在外观设置中，选择“宠物”。

然后领养一个自己喜欢的宠物即可。

总结

网上有些文章提到，可以通过终端命令开启 Codex 宠物。但我实操下来，直接在终端中输入 /pet 没有效果。

也许是我没有找到正确的打开方式。至少目前来看，通过 Codex 桌面 App 进入设置，再从外观里开启宠物，是一条可行的路径。
