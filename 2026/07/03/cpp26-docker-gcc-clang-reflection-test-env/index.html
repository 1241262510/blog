C++26 测试环境搭建：用 Docker 跑 GCC 16 和 Clang-p2996

C++26 已经可以在一些较新的编译器和实验分支里试用了。日常学习时，我更倾向于用 Docker 准备环境：不污染本机，也方便在不同编译器之间切换。

这篇文章记录三种用法：

  1. 本机 GCC 15：测试基础 C++26 语法。
  2. Docker GCC 16：测试 GCC 对 C++26 和反射的实现。
  3. Docker Clang-p2996：测试 P2996 反射实验分支。

需要注意：C++26 反射相关实现仍在快速变化，下面的命令更适合作为“测试环境搭建记录”，不是长期稳定 API 保证。如果编译器镜像更新后报错，优先检查镜像版本和反射编译参数。

六、编译参数速查

场景 命令或参数
本机 GCC 15 g++ -std=c++26 test.cpp -o test
Docker GCC 16 docker run --rm -v "$PWD/test.cpp:/test.cpp" gcc:16 sh -c 'g++ -std=c++26 -freflection /test.cpp -o /test && /test'
Docker Clang-p2996 clang++ -std=c++26 -freflection-latest -stdlib=libc++ -fexpansion-statements

小结

如果只是学习 C++26 基础语法，本机 GCC 15 已经足够做很多实验。如果要测试反射，建议用 Docker 单独准备 GCC 16 或 Clang-p2996 环境，避免把实验性编译器和本机开发环境绑在一起。
