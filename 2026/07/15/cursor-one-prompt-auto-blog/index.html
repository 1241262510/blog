Cursor 一句话生成 Blog：免费 Auto 模型能做到什么程度？

很久以前，我安装并体验过 Cursor。那时整个过程很简单，下载、安装、登录，几分钟后就能开始使用。

最近我重新安装了一遍，发现注册流程与以前相比多了一道手机验证。我的英国 giffgaff 手机卡可以正常接收验证码，最终顺利完成了验证。需要说明的是，手机验证似乎并非所有账号都会遇到，更像是系统根据账号、网络环境等情况触发的额外验证。

完成注册后，我想测试一个很直接的问题：不购买 Cursor Pro，只使用免费版，能不能靠一句话生成一个像样的个人博客？

免费版还能不能用？

答案是：能用，但选择会受到限制。

免费账号仍然可以使用 Cursor 的 Agent 和 Auto 模式，让它读取项目、创建文件并修改代码。不过，在免费额度和模型权限下，通常不能像付费用户那样自由选择 Claude、GPT 等热门模型。这里容易混淆的一点是，Claude Code 是 Anthropic 的命令行编程工具，并不是一个模型名称；Cursor 中实际选择的是 Claude、GPT、Gemini 等模型或 Auto 自动路由。

Auto 模式的好处是不用纠结具体选哪个模型。Cursor 会根据任务和当前可用资源自动处理请求。代价也很明显：模型选择过程不透明，输出效果不够稳定，也很难判断某一次任务究竟由哪个模型完成。

一句话生成一个博客

这次测试没有写复杂的需求文档，也没有提前设计页面。我只给了 Cursor 一个大致目标，让它生成一个可以在本地运行的个人 Blog。

Cursor 随后自动创建了项目结构、页面样式和基础内容。整个过程不需要我逐个新建文件，也不需要从零手写页面。对于不熟悉前端开发的人来说，这种体验确实很有吸引力：一句话描述想法，很快就能看到一个可以运行的成品。

最终生成的页面能够正常打开，基本布局也完整，包括标题、正文区域以及页脚：

© 2026 随笔 · 运行在本地

从“有没有做出来”这个角度看，Auto 模型完成了任务。它不是只给出一段示例代码，而是搭出了一个真正能够运行的博客页面。

成品体验：像一套“框架房”

如果只看第一眼，生成结果还算不错：页面干净，结构清楚，没有明显的错位，作为演示项目完全够用。

但从设计与功能的角度仔细体验，它与成熟博客还有明显距离。和我之前用 Grok 生成的 Blog 相比，这次 Cursor 生成的版本更像一个基础框架，完成度没有那么高。

主要问题体现在几个方面：

  • 页面视觉层次比较单一，字体、留白和配色缺少细致打磨；
  • 博客只有基础展示能力，缺少文章列表、分类、标签和搜索等常用功能；
  • 没有针对手机端进行充分优化；
  • 缺少深色模式、代码高亮、文章目录和阅读进度等体验细节；
  • 示例内容较少，看起来更像 Demo，而不是可以长期维护的网站。

我愿意把它形容成一套“框架房”：主体结构已经搭好，水电也能用，确实可以住人，但谈不上舒适。想把它变成一个真正好用的博客，还需要继续装修。

一句话生成，不等于一句话交付

这次体验也让我更明确地感受到：AI 很擅长把项目从 0 推到 1，却很难只靠一句模糊的要求，就把产品从 1 打磨到 100。

如果继续使用 Cursor 完善这个博客，下一步至少还需要提出更具体的要求，例如：

  1. 统一字体、颜色、卡片和间距，建立完整的视觉风格；
  2. 增加文章列表、分类、标签、搜索和归档页面；
  3. 加入响应式布局，分别检查电脑和手机端效果；
  4. 完善 Markdown 渲染、代码高亮、目录与上一篇/下一篇导航；
  5. 增加深色模式、SEO 信息、RSS 和站点地图；
  6. 准备真实文章，再根据内容长度调整版式。

换句话说，第一句提示词负责“把房子盖起来”，后续的一轮轮交流才是在“装修房子”。提示词越具体，验收标准越清晰，AI 生成的结果通常也越接近预期。

总结

Cursor 免费版目前依然有体验价值。即使不能自由调用热门模型，Auto 模式也足以完成简单页面、原型项目和基础博客的搭建。对于想快速验证创意，或者刚开始接触 AI 编程的人，它能够显著降低从想法到成品的门槛。

不过，如果目标是做一个设计成熟、功能完整、可以长期使用的 Blog，一句话远远不够。Auto 模型可以替你搭好框架，但审美判断、功能取舍和最后的细节打磨，仍然需要人来完成。

这次测试给我的结论是：Cursor 一句话确实能生成 Blog，但生成的是起点，不是终点。
