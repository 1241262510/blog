用 Obsidian 打通公众号创作工作流

为何选择 Obsidian

选择这款应用来打通公众号创作工作流，主要有以下原因：

  • 原生 Markdown 支持：Obsidian 提供了完善的 Markdown 写作功能，数据可以在不同平台间自由流转。虽然 mdnice、MarkCopy、MarkText 等工具也支持 Markdown，但它们的数据是孤立的，缺乏扩展能力。

  • 插件生态丰富：Obsidian 拥有开放的插件市场，这为应用插上了扩展的翅膀，让功能无限可能。

  • AI Agent 全面支持：目前全面拥抱 Agent 的软件并不多，国内的飞书算一个。其他软件由于系统封闭或其他原因，对 Agent 的支持有限。Obsidian 天生的架构设计使其对 Agent 的支持水到渠成，几乎第一时间就能支持 OpenClaw 等工具。这为喜欢终端操作环境的用户提供了极大便利。

终端工作流与 Obsidian 的差异

目前我已经实现了在终端使用大模型进行公众号和 GitHub 的同步，基本达成了在公众号和 GitHub 发布博文的目标。

借助大模型，我已经打通了博文写作和发布的工作流。

实操起来，难度不高。主要问题是获取公众号的 AppID 和密钥，并在配置环境中部署这些参数，然后借助终端进行同步和发布。

对我个人而言，比较喜欢对话式的更新和发布方式。但对于不习惯终端操作的用户，可能需要时间去适应。

Obsidian 则可以搭建一个可视化操作环境，用于博文的发布和更新。

写作、发布全程无需使用终端，对普通用户来说，体验会更好。

普通用户可能更喜欢这样的发布模式。

今天，就为大家介绍这个工作流的使用流程和操作步骤。

部署前提

  • 获取微信公众号的 AppID 和 Key
    无论是使用终端还是 Obsidian，获取 AppID 和 Key 都是必要条件。没有这些参数，无法实现公众号的自动化工作流。

  • 插件安装
    微信公众号的发布有几种插件可以选择，今天使用其中一个插件进行工作流的测试安装。

  • Obsidian 基础使用
    由于是在 Obsidian 应用内使用，下载和基础操作都在这个平台上进行。因为本文的重点是介绍工作流，所以 App 的下载和基础安装就不赘述了，大家可以参考网上的其他教程。

AppID 和 Key 的获取

这是腾讯公众号参数的获取路径。

自动化发布流程需要的参数可以从这个网址获取。如果没有注册个人小程序，需要先注册，才有权限获取这些参数。

进入微信开发者平台，点击 账号管理，然后 前往公众号详情页

在 开发密钥 处进行设置

IP 白名单 也需要设置

可以通过 what is my ip 等网站获取你的 IP 地址。

取得这些参数后，就可以在对应的 Obsidian 插件中进行设置。

Obsidian 插件设置

先说说部署流程，熟悉了一个插件的部署流程后，其他插件的操作流程几乎大同小异。

进入网址后，下载对应的插件

在本地环境解压，然后拷贝到 Obsidian 对应的插件目录中

在 macOS 系统中，使用 Shift + Command + . 打开隐藏文件显示开关，即可看到我们需要的 plugins 目录，将下载解压的文件夹复制到这个目录中。

在 Obsidian 中激活插件

在 App 中找到设置

找到 第三方插件 选项，打开开关进行激活。如果没有这个选项，需要先关闭 安全模式，然后重新激活开关。

参数设置

将获取的 AppID 和密钥填写到对应的参数中。

文章发布和预览

使用 Command + P 搜索插件，找到我们安装的插件

编辑页面与插件页面共存的布局。可以看到主要功能包括传统的复制/粘贴选项，以及直接发布的选项（草稿）。

与使用 mdnice 等网站类似，这里也解决了 Markdown 文档与微信 HTML 之间的格式转换，效果是一样的。

经过测试，直接复制粘贴就能实现平时在其他网站进行的操作。

第二个选项就是我们所说的自动化发布。

我测试使用”复制到公众号”功能，没有任何问题。但使用”发布”功能时，会出现图片丢失的问题。

借助大模型的帮助，可以很好地解决这个技术问题。有需要的可以留言，我会把修改版本同步到 GitHub 博客中。

总结

这个工作流，可能更符合普通用户的”审美”，更容易操作和上手。
