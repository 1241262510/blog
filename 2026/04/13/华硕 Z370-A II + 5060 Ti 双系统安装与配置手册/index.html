华硕 Z370-A II + RTX 5060 Ti 双系统安装与配置手册

摘要

这篇文章记录一套典型、也很容易踩坑的装机组合：华硕 Z370-A II 主板、Intel Xeon E-2136 处理器、16GB 内存、RTX 5060 Ti 显卡和 1TB NVMe SSD。

看起来只是一次普通的双系统安装，但真正麻烦的地方并不在安装包本身，而在于老平台 BIOS 习惯、UEFI 启动方式、显卡资源映射和新显卡之间的兼容关系。

如果 BIOS 仍然保持默认状态，Ubuntu 安装器在配置引导阶段报“脚本错误”，Ventoy 下的 Win11 安装也可能不稳定。这类问题很多时候不是系统本身坏了，而是底层启动和资源分配条件没准备好。

一、硬件环境回顾

先把这台机器的基础配置列清楚：

  • 主板：华硕 Z370-A II，LGA1151 平台，Z370 芯片组
  • CPU：Intel Xeon E-2136，6 核 12 线程，没有核显
  • 内存：16GB DDR4
  • 显卡：NVIDIA RTX 5060 Ti，支持 AV1、DLSS 4.0+，对显存地址空间要求更高
  • 存储：1TB NVMe SSD

这套配置的核心特点是“老平台 + 新显卡”。CPU 和主板属于较早一代的桌面平台，但显卡已经是更新一代的产品，二者之间并不总是天然兼容。

二、核心问题：新显卡和老主板的“脚本错误”

安装 Ubuntu 时，程序可能在配置引导或者检测显卡阶段抛出“脚本错误”；用 Ventoy 安装 Win11 时，也可能出现不稳定、卡死或者反复回退。

表面上看，这是安装器的问题；实际上，根因往往是 BIOS 没有切到适合新显卡和 UEFI 启动的模式。

5060 Ti 这一类新显卡，对以下两个条件更敏感：

  • UEFI 启动链路是否完整
  • 显存映射和 PCIe 资源分配是否足够

如果 BIOS 仍然保留 CSM 兼容模式，或者没有开启 Above 4G Decoding、Re-Size BAR 这类选项，安装阶段就可能因为资源冲突而出错。

三、终极破解：BIOS 深度优化清单

进入华硕 BIOS 的 Advanced Mode (F7) 后，建议逐项确认下面这些设置。

A. 解决“脚本错误”的关键：Boot 设置

  • Launch CSM 设为 Disabled

  • 说明：必须关闭。对这类新显卡来说，CSM 往往只会带来额外的兼容冲突。

  • Secure Boot 中的 OS Type 设为 Other OS

  • 说明：这样可以减少安全启动策略在安装 Linux 或第三方启动介质时带来的限制。

B. 解决“显卡资源冲突”：Advanced 设置

  • Above 4G Decoding 设为 Enabled

  • 说明：这是关键项。新显卡对 PCIe 资源和内存映射空间要求更高，不开启时，安装过程可能直接卡死或报错。

  • Re-Size BAR Support 设为 Enabled

  • 说明：让 CPU 更直接地访问显存，不只是游戏性能会受益，后续开发和计算场景也会更稳。

C. 解决 Win11 和硬盘检测问题：System 设置

  • PTT（Firmware TPM）设为 Enable

  • 说明：用于满足 Win11 的 TPM 检测要求。

  • SATA Mode 设为 AHCI

  • 说明：不要用 RAID。对 Linux 来说，AHCI 更稳定，也更不容易出现找不到盘的情况。

C. 双系统共存建议

双系统环境下，建议把下面几件事固定下来：

  • Windows 和 Ubuntu 都使用 UEFI 启动
  • 不要混用传统 Legacy 启动方式
  • 系统盘尽量保持单一分区风格，不要在安装过程中反复切换启动模式
  • 每次改 BIOS 后，先确认能否稳定识别 NVMe SSD

五、安装过程里的实际经验

这类机器最容易出问题的，不是系统镜像坏了，而是“看起来都对，实际上差一个关键选项”。

我的建议是按这个顺序排查：

  1. 先确认 CSM 已关闭
  2. 再确认 Above 4G Decoding 和 Re-Size BAR 已开启
  3. 然后把 PTT 和 AHCI 状态检查一遍
  4. 最后再启动安装介质，重新跑 Ubuntu 或 Win11 安装流程

如果 BIOS 调整正确，安装器通常会明显稳定很多。很多“脚本错误”本质上不是脚本，而是底层资源条件不满足。

六、总结

这套组合的结论很简单：老主板不是不能带新显卡，但不能继续沿用老平台默认思路。

对华硕 Z370-A II + RTX 5060 Ti 这类组合来说，真正决定成败的通常是 BIOS 里的几个开关：

  • 关闭 CSM
  • 打开 Above 4G Decoding
  • 打开 Re-Size BAR
  • 配好 PTT
  • 保持 AHCI

如果这些都处理好了，Ubuntu 22.04 和 Win11 的双系统安装会顺畅很多，后续再做驱动和工作流配置也会省力得多。

对我来说，这类折腾最有价值的地方，不是“终于装上了”，而是把新硬件和旧平台之间的边界摸清楚了。

七、收尾说明

如果你也在用类似的老平台配新显卡，最值得优先确认的不是系统版本，而是 BIOS 里的启动方式、资源映射和 TPM 状态。

一旦这三类基础条件稳定下来，后面的 Ubuntu 安装、Win11 安装、显卡驱动和日常使用都会轻松很多。
